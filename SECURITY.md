## Reporting a Vulnerability

We take the security of AFK seriously. If you believe you have found a security vulnerability in AFK, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

To report a vulnerability, please contact us via one of the following methods:

1.  **Email**: [contact@arpan.sh](mailto:contact@arpan.sh) (Preferred; **Mandatory for highly sensitive vulnerabilities**)
2.  **LinkedIn**: Message the maintainer directly.
3.  **GitHub**: If available, use the "Report a vulnerability" tab in the Security section of the repository. **Do NOT use this for highly sensitive vulnerabilities.**

Please include as much information as possible to help us reproduce the issue, including:

- The version of AFK you are using.
- A description of the vulnerability.
- Steps to reproduce.
- Impact of the vulnerability.

### Response Timeline

We will acknowledge your report within 48 hours and provide an estimated timeline for a fix. We will keep you updated on our progress.

### Acknowledgement

We appreciate your efforts to improve the security of AFK. We will acknowledge your contribution in our release notes and security advisories, unless you prefer to remain anonymous.
